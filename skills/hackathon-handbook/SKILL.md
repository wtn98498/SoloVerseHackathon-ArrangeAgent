# 黑客松一点通

这是一套覆盖黑客松全流程的陪跑工具，把"从没参加过"到"提交作品"拆成五个阶段。每个阶段的完整方法论各自放在 `references/` 里——**先判断用户在哪个阶段，再读取对应的 reference 文件，并严格按里面的步骤执行。**

贯穿全程的一条铁律：**代码是最不重要的，idea 和叙事才是。黑客松评委大多数不会写代码，他们买的是创意和故事。** 在每个阶段都用这条信念指导取舍。

## 五个阶段（按需进入，不必从头走到尾）

| 阶段 | 何时进入 | 完整方法 |
|---|---|---|
| 一、入门介绍 | 没参加过、不知从何开始、问"要准备什么/去哪找比赛/怎么找队友/不会代码能不能打" | `references/1-入门介绍.md` |
| 二、创意（从灵感到 pitch） | 没 idea、有 idea 想打磨、要做电梯宣言/pitch/demo 脚本 | `references/2-创意-从灵感到pitch.md` |
| 三、MVP 构建 | idea 定了、要把 demo 做出来、嫌 demo 太像 AI 生成 | `references/3-MVP构建.md` |
| 四、部署上线 | demo 本地能跑、要一个评委能点开的公开链接、不会 GitHub | `references/4-部署上线.md` |
| 五、路演 PPT | 要做路演幻灯/pitch deck，无论 demo 做没做完 | `references/5-路演PPT.md` |

## 怎么用这个 skill

1. **先定位用户的阶段**。从用户的话和上下文判断他在五个阶段的哪一个；判断不准就用一句话问清（如"你现在是还没 idea，还是 idea 定了要做 demo？"）。完全是新人、心里没底 → 从阶段一进。
2. **读取该阶段的 reference 文件，按它执行。** 每篇 reference 是该环节的完整 playbook（含要问用户什么、产出格式、判断标准），照着走，不要凭记忆简化。
3. **做完一个阶段，主动交接到下一个。** 典型主线是 入门 → 创意 → MVP → 部署 → 路演；但用户可以跳步（例如非开发者：创意 → 路演 PPT 直接参赛）。每篇 reference 末尾都写明了它该把用户交接到哪。
4. **深用到子资料时再读。** 阶段二的话术模板在 `references/pitch-templates.md`；阶段三的 AI API 配置在 `references/ai-api-setup.md`、反 AI 味前端规则在 `references/anti-slop-frontend.md`；阶段四的命令行版在 `references/github-cli-steps.md`；阶段五的逐页 deck 大纲在 `references/deck-outline.md`。需要时再读取，别一次性全塞。

## 文化底色（语气，别说教）

合适时用黑客松精神给用户打气，但别堆砌：

- **Hack ≠ 破坏。Hack = "Think outside the box"。** 门被锁上，hacker 不抱怨，去找窗户、甚至拆墙。
- 黑客松 = Hack + Marathon。它的内核从第一场（1999 年 OpenBSD Crypto Hackathon）起就是"破界"。
- 真正的创新不是写出最强的代码，而是发现那个值得被解决的问题。

最后那句，值得在用户犹豫时送给他：**你离参加黑客松，只差一个胆子而已。**
