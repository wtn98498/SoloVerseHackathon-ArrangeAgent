# 反 AI 味前端规则（提炼自 Taste Skill / tasteskill.dev）

写任何 UI 之前读这份，并在交付前过一遍发布检查清单。目标：让一个黑客松 demo 看起来像有设计师参与，而不是"AI 默认生成"。

完整且持续更新的版本可在项目里安装：`npx skills add Leonxlnx/taste-skill`。下面是离线可用的核心。

## §0 先读懂"这是什么"，再动手

生成任何界面前先确定五件事，写下来：
- **行业 / 受众**：B 端工具？消费产品？给老人用？给开发者用？
- **气质（mood）**：冷静专业 / 温暖亲和 / 锋利极简 / 粗野有力。
- **动效深度**：克制（仅过渡）还是有表现力（滚动叙事）。
- **布局家族**：编辑式、仪表盘、落地页、卡片流——选一个，别混。
- 一两个**真实参考**（不是"现代简约"这种空话，而是具体的网站气质）。

方向定了再写代码。不要让模型用默认审美自由发挥——那就是 AI 味的源头。

## 必须避免的"AI 味八件套"（硬禁止）

1. 紫色→蓝色的对角线渐变背景。
2. 所有东西都居中：超大居中标题 + 居中副标题 + 居中按钮。
3. 千篇一律的圆角卡片三件套（三个等宽卡 + emoji 图标 + 一句话）。
4. 默认只用 Inter / 默认系统字体且不做任何字号层级设计。
5. 通篇 emoji 当图标用。
6. 每个区块配色、间距、节奏完全一样，从上到下像复制粘贴。
7. 毫无意义的玻璃拟态 + 霓虹光晕堆砌。
8. 按钮一律是"实心圆角 + 轻微阴影"的同一种。

看到自己要写出以上任意一条，停下来换做法。

## 排版（最快拉开质感的地方）

- **建立明确的字号层级**：标题、副标题、正文、说明文字之间要有清晰跨度（如 clamp 大标题 vs 14–16px 正文），不要全是 16–20px。
- **选一对有性格的字体**：标题用有辨识度的字体（衬线 / 几何无衬线 / 展示体），正文用易读字体。中文 demo 注意选好中文字体回退，别让中文掉进默认宋体。
- 行高、字距按用途调：大标题收紧，正文放松。
- 一屏内别超过 2 种字体家族。

## 颜色

- 选一个**真正的主色**（不是默认蓝/紫），加 1 个强调色，其余用中性灰阶。
- 用足够的中性色和留白撑起层级，颜色用在"该被看见"的地方。
- 暗色模式不是把背景改黑——要保证对比度和层级在两种模式下都成立（如果做暗色）。

## 布局与间距

- **打破对称**：用非对称布局、不等分网格、错落的留白制造节奏，不要每块都居中。
- 间距成体系（4/8 的倍数），区块之间留白要慷慨。
- 不同区块要有**节奏变化**：有的密、有的疏；有的窄、有的通栏。
- 对齐要严格——边缘对齐是"专业感"的隐形来源。

## 动效

- 克制优先：进入过渡、hover 反馈、关键状态变化做平滑即可。
- 动效服务于"那个记得住的瞬间"——把表现力集中在 demo 的高光处，别全站乱动。
- 别用浮夸但无意义的视差和粒子。

## 细节（体现尊重，也容易拿分）

- 真实内容，不要 "Lorem ipsum" 和占位图堆砌——demo 用贴近真实的文案和数据更可信。
- 状态齐全：空状态、加载、hover、focus 至少把出现在 demo 里的那几个做好。
- 针对受众调可用性：给老人用就字大、对比强、交互简单（重新审视现代 App 对目标人群是否"侵犯性"）。
- 圆角、阴影、边框三选一克制使用，不要三个一起堆满。

## 发布前检查清单（demo 录制前逐条过）

- [ ] 有没有踩"AI 味八件套"里的任何一条？踩了就改。
- [ ] 字号层级是否明显（标题和正文一眼能分清）？
- [ ] 主色是不是刻意选的，而不是默认蓝/紫？
- [ ] 布局是否打破了"全部居中"，有非对称和节奏？
- [ ] demo 里实际会出现的状态（空/加载/hover）是否都正常？
- [ ] 中文字体是否正常、没掉默认宋体？
- [ ] 在真实尺寸（评委会看的屏幕/手机）上看过一遍？
- [ ] "那个记得住的瞬间"是不是最精致、最被强调的部分？

任何一条没过，回去改再录。
