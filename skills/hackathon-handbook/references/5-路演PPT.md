# 路演 PPT

把项目压成一份评委 3 分钟内能看懂、记得住的 deck。黑客松的 PPT 不是产品文档，是 pitch 的视觉版——评委大多不写代码，他们看的是问题、故事、和那个"记得住的瞬间"。**简单、克制、有一个高光，胜过塞满信息。**

## 先确认触发时机（决定第 demo 页怎么做）

问用户（或从上下文判断）现在处于哪个阶段：

- **A. idea / pitch 定了，demo 还没做** → 做"概念版 deck"。demo 那一页用一张概念图 / 手绘草图 / mockup 占位，标注"概念演示"。适合先准备、或非开发者直接用 PPT 参赛。
- **B. demo 已经做完** → 做"完整版 deck"。demo 那一页放真实截图 + 线上链接（来自部署阶段的 vercel.app URL），并留一句话引导现场切到 live demo。

两种只差 demo 那 1–2 页，其余结构相同。

## 先取材，再做页（不要从空白开始）

动手前，先把创意阶段的产出捞齐（没有就先快速补，或引导用户回去用创意阶段）：

- **电梯宣言**（15 秒那句）→ 封面副标题 + 结尾页。
- **根问题 + 场景化叙事里的那个真人**→ 问题页。
- **解决方案一句话**→ 方案页。
- **那个"记得住的瞬间"**→ demo 页的重点。
- 主办方 / 赞助商技术（若有）→ 在方案或技术页点出关联性。

## 标准结构（6–8 页，简单为先）

按这个骨架，每页一个核心信息，别堆：

1. **封面**：项目名 + 一句话电梯宣言（不是 slogan 堆砌）+ 队名/作者。
2. **问题**：那个真人的具体痛点场景，点出"根问题"。一张有冲击力的图或一个大数字，不要满屏字。
3. **解决方案**：一句话讲清你怎么解决 + 一张产品示意。避免技术术语。
4. **Demo**：
   - 时机 A：概念图 / mockup + "概念演示"标注。
   - 时机 B：真实截图（1–2 张）+ 线上链接 + "现场切 live demo"提示。
   - 把"记得住的瞬间"放在这页最显眼处。
5. **它怎么运作 / 技术亮点（可选，1 页）**：极简。用到的赞助商技术、AI 能力一句话带过。评委不关心架构。
6. **影响力 / 想象空间**：往大里说一句可能性（衍生模式、人群规模、社会价值）。
7. **结尾页**：回扣电梯宣言的金句 + 链接/二维码 + 联系方式。

非开发者或时间紧时，砍到 5 页（封面/问题/方案/概念演示/结尾）完全够用。

## 设计规则（让 deck 不像 AI 生成）

- **选一个贴合主题的配色**，不要默认蓝。一个主色占 60–70%，1 个强调色，其余中性。可参考：午夜行政 `1E2761`+`CADCFC`、珊瑚活力 `F96167`+`F9E795`+`2F3C7E`、暗黑高级（深底白字）。标题/结尾用深底，内容页浅底（"三明治"结构）。
- **字号要有层级**：标题 36–44pt 加粗，小标题 20–24pt，正文 14–16pt。标题和正文一眼能分清。
- **每页都要有一个视觉元素**（图/图标/大数字/形状），不要纯文字 + 项目符号。
- **正文左对齐**，只有标题居中。
- **绝对不要在标题下面加装饰性横线**——那是 AI 生成 PPT 的典型特征，用留白或背景色区隔。
- 选一对有性格的字体配对（如标题 Georgia / Arial Black，正文 Calibri）。中文注意字体回退，别掉默认宋体。
- 留白慷慨，0.5" 以上页边距，别填满每一寸。

## 生成方法

1. **如果当前环境里有 pptx / PowerPoint 相关 skill，优先调用它**来生成（它有完整的模板和 QA 流程）。
2. **如果没有**（如纯净的 Claude Code 环境），用 pptxgenjs 自己生成，最稳：

```bash
npm install -g pptxgenjs   # 或在项目里 npm install pptxgenjs
```

按 `references/deck-outline.md` 里的逐页内容映射和一段最小 pptxgenjs 示例来写脚本，`node make_deck.js` 生成 `pitch.pptx`。

## 生成后必做 QA（别跳过）

- 把每页导成图片看一遍（`soffice --convert-to pdf` 再 `pdftoppm -jpeg`），逐页找问题：文字溢出/重叠、对比度太低、留白不均、标题下有没有误加横线。
- 检查有没有占位符残留（"xxx"/lorem/TODO）。
- 念一遍封面副标题，是不是就是那句 15 秒电梯宣言。
- 时机 B 确认链接能点开、截图清晰。

发现问题就改了再交，别一版就收。

## 交接

- 时机 A 做完概念 deck → 提示用户："要把 demo 真的做出来吗？用 **阶段三·MVP构建**（`references/3-MVP构建.md`）。做完可以回来更新这份 deck 的 demo 页。"
- 时机 B 做完完整 deck → 这套东西就齐了：idea、叙事、pitch、可点 demo、路演 PPT。提示用户对着 `references/pitch-templates.md` 里的 pitch 结构排练一遍 3 分钟讲稿。
