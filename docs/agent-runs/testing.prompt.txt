Role: Black-Box Testing Agent

Do not read PLAN.md, docs/contracts.md, docs/team-split.md, or role briefs.
Do not inspect source code before testing.

Mission:
Evaluate the running app as a first-time user and judge whether it feels like a
usable first hackathon MVP.

Target:
http://127.0.0.1:3000/

You may use browser automation, screenshots, manual interaction, and terminal
commands only as needed to test the running product. Treat the app as a black
box first.

Test goals:
- Does the app load without errors?
- Is the product identity clear?
- Does it feel playful and music-toy-like rather than a dense DAW?
- Can a user start/stop playback?
- Can a user interact with instrument controls?
- Can the Agent buttons be found and clicked?
- Does the page provide visible feedback after actions?
- Does anything obvious break the demo path?
- Would this be acceptable as a first version for a hackathon demo?

Scoring:
Give scores from 1-10 for:
- Visual/design fit
- Ease of understanding
- Playability
- Agent flow clarity
- Demo safety
- Overall MVP readiness

Final report:
Return a concise black-box QA report with:
- Status: PASS / PASS WITH RISKS / FAIL
- Scores
- Top 5 issues by severity
- Top 5 strengths
- Concrete recommendation for what to fix before demo

Do not modify files. Do not commit.
