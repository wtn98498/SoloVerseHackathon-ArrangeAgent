# 01_cover

大家好，我是温添宁，参赛假名 scala，队伍叫欧比克莱歌剧院。我们的项目是 PlayBand AI。它解决的是一个很简单但很常见的问题：很多人有音乐感觉，但不会乐理，也不会用专业 DAW。所以我们的核心承诺是，先玩，再编曲。用户先敲出一点灵感，AI 音乐导演再把它扩展成一支乐队。

# 02_problem

今天普通人做音乐，通常卡在两个入口之间。专业 DAW 很强，但一打开就要求你理解轨道、插件、钢琴卷帘，这对新手太重。另一边，一句话生成歌曲很轻松，但人的手感被跳过了，结果不像是自己做出来的。我们认为中间缺了一类工具：让用户先玩一下，再让 AI 把这点感觉放大。

# 03_insight

PlayBand AI 的产品洞察是：灵感种子应该来自人，编曲扩展可以交给 Agent。人负责品味、节奏感和身体上的直觉，Agent 负责结构、配器和变化。最关键的是，结果不是一个黑箱音频，而是看得见、听得到、还能继续改的音乐结构。

# 04_solution

具体怎么做？用户用大 pad 敲几下，系统把这些敲击捕捉成轻量 MIDI-like 事件。然后音乐导演把这段种子补成 8 小节 loop，包含鼓、贝斯、吉他和键盘。生成结果不会直接覆盖作品，而是先变成候选，用户试听满意以后再应用。这样 AI 是副驾驶，不是替你抢方向盘。

# 05_demo

这页是我们希望评委记住的一句话：我敲了几下，它变成一支乐队。现场演示时我会打开链接，先敲一个短鼓点，把它捕捉进 MIDI-like 数据，再点补全编曲。你会看到候选编曲出现，并且可以试听、应用，然后继续用自然语言说更快一点，或者柔和一点。这就是完整闭环。

# 06_architecture

技术上，我们没有把项目做成脆弱的 AI 表演。前端是 React 和 TypeScript，负责可玩的编辑器、pad、候选卡片和信任视图。音乐播放用 Tone.js。核心数据是 8 小节、128 步的轻量 MIDI-like JSON，所以每次 Agent 输出都能被显示、播放和验证。即使没有模型 key，本地音乐导演 fallback 也能稳定完成 demo；之后 DeepSeek 可以接在同一套动作边界后面。

# 07_why_win

我觉得这个项目适合黑客松，是因为它不是堆功能，而是打中了一个可以被复述的产品瞬间。它好记：几下敲击变成一支乐队。它可信：生成的是结构化音乐数据。它稳定：本地 fallback 保护现场演示。它也足够聚焦：只做从灵感种子到 8 小节 band loop 的核心路径。

# 08_future_close

未来我们会在同一个候选预览循环里接入更强的模型编曲能力，增加不同风格包，比如 lo-fi、rock、pop，以及古风和民族乐器包。等核心 loop 稳定以后，再支持 MIDI 和 stems 导出，甚至让社区里的小 riff 变成可以 remix 的创作起点。最后回到一句话：先玩，再编曲。谢谢大家。
