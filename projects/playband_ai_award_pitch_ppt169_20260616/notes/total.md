# 01_cover

大家好，我是温添宁，参赛假名 scala，队伍叫欧比克莱歌剧院。我们的项目是 PlayBand AI。它解决的是一个很简单的问题：很多人有音乐感觉，但不会乐理，也不会用专业 DAW。我们的核心承诺就是这一句：先玩，再编曲。用户先敲出一点灵感，AI 音乐导演再把它扩展成一支乐队。

# 02_toc

今天我只讲一件事：普通人的一小段灵感，怎么被 Agent 编成歌。接下来会分四步走。先讲背景和灵感来源，再讲产品怎么让人先玩起来，然后重点展示我们的音乐导演 Agent，最后讲未来规划和可以继续扩展的方向。

# 03_inspiration

这个项目的灵感来自三个地方。Cursor 让我意识到，Agent 最好的位置不是替人完成一切，而是理解人的意图，再补上结构。GarageBand 让我看到音乐工具可以很有玩具感，但它对新手还是有门槛。现在很多 AI 生成歌工具很轻松，可是人的手感被跳过了。所以我想做的是音乐版 Cursor：人先表达直觉，Agent 再把它变成可继续修改的作品。

# 04_problem

今天普通人做音乐，通常卡在两个入口之间。专业 DAW 很强，但一打开就要求你理解轨道、插件、钢琴卷帘，这对新手太重。另一边，一句话生成歌曲很轻松，但结果不像是自己做出来的。中间缺的是一个入口：让用户先玩一下，再让 AI 把这点感觉放大。

# 05_insight

PlayBand AI 的产品洞察是：灵感应该来自人，编曲可以交给 Agent。人负责品味、节奏感和身体上的直觉，Agent 负责结构、配器和变化。界面负责把结果变成看得见、听得到、还能继续改的音乐结构。这样 AI 不是黑箱，而是创作副驾驶。

# 06_solution

这一页讲技术架构。PlayBand AI 不是直接生成一段音频，而是把用户的手感先变成结构化数据。前端用 React 做一个玩具式音乐编辑器，大 pad 捕捉用户的敲击、风格和情绪；这些输入会被保存成轻量 MIDI-like 的 SeedPattern。然后前端调用 Agent API，例如 complete arrangement 或 energy transform，服务层返回一个 ArrangementProject 候选。候选不会直接覆盖当前作品，用户可以先试听，再决定应用、重来或者继续修改。最后 Tone.js 按这个结构化工程播放，所以评委能同时看到轨道变化，也能听到声音变化。

# 07_demo

这页是我们希望评委记住的一句话：我敲了几下，它变成一支乐队。现场演示时我会打开链接，先敲一个短鼓点，把它捕捉进 MIDI-like 数据，再点补全编曲。你会看到候选编曲出现，并且可以试听、应用，然后继续用自然语言说更快一点，或者柔和一点。这就是完整闭环。

# 08_agent_director

这一页讲 Agent 编曲原理。它不是黑箱音频生成器，而是结构化编曲器。第一步是保留用户 seed，不把用户刚刚敲出来的节奏感抹掉。第二步根据风格、情绪和速度推导一个很小的音乐骨架，例如和弦走向、密度和能量。第三步分别写四个轨道：鼓负责 downbeat 和 backbeat，贝斯跟随根音并加入简单 passing note，吉他和键盘负责和弦、riff 和空间感。第四步输出 ArrangementProject 候选，并做 step、pitch、velocity 校验。当前提交版用本地确定性编曲引擎保证 demo 稳定；DeepSeek 会接在同一个 API 合约后面，负责更开放的风格理解和编曲决策。

# 09_future

未来我们会在同一个候选预览循环里接入更强的模型编曲能力，让音乐导演更懂风格、更懂用户的 taste。之后可以增加 lo-fi、rock、pop，以及古风和民族乐器包。等核心 loop 稳定以后，再支持 MIDI 和 stems 导出，甚至让社区里的小 riff 变成可以 remix 的创作起点。

# 10_thanks

最后回到这句话：先玩，再编曲。PlayBand AI 想做的不是让普通人绕过创作，而是把他们的音乐直觉留在创作循环里。谢谢大家，我是温添宁，队伍是欧比克莱歌剧院，欢迎现场体验 PlayBand AI。
